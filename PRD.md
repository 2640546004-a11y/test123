# 产品需求文档 (PRD) - 树洞自律小刺猬 Web MVP 3.0

## 1. 产品概述
* **产品定位**：本产品是一款面向 PC 和 Mac 用户的极简、零负担桌面数字疗愈自律工具[cite: 6]。
* **核心理念**：产品致力于拒绝生硬的“防沉迷锁屏”机制。通过“帮助小动物回家”的叙事体验、动态声音陪伴以及“自我慈悲”的温柔反馈，旨在激发用户的内在专注动力[cite: 6]。
* **目标用户**：主要面向需要长时间专注且面临数字焦虑的年轻用户群体（考研/考公学生、程序员、自由职业者等）[cite: 6]。

## 3. 产品原型及页面逻辑说明

### 3.1 起跑仪式页（产品首页）
* **页面入口**：用户直接访问产品 Web 端 URL，无其他前置入口[cite: 6]。
* **背景与声音渲染**：
  * 页面加载完成后，后台自动静音循环播放 `assets/videos/home-bg.mp4`[cite: 6]。
  * **【新增音轨】**：同步**自动循环播放首页背景音乐 `assets/audio/home-bgm.mp3`，音量在代码层严格限制在 `0.25` (25%)**。
* **核心交互**：
  * **【新增按键音效】**：为全局所有可点击的元素（时间轴数字、START 按钮、邮箱图标）统一绑定监听。**在鼠标按下（click）的瞬间，必须触发播放一次短促清脆的音效 `assets/audio/click.mp3`，音量设定为 25%**。
  * **时间选择**：用户点击顶部木质时间轴，默认选中 25 分钟[cite: 6]。选中状态文字亮白并带暖黄微光，切换时长平滑无错位[cite: 6]。
  * **START 按钮**：点击瞬间触发点击音效，首页 UI 元素在 1.0 秒内平滑淡出，同步调用浏览器 Fullscreen API 拉起全屏，无缝切入【全屏禅定页】[cite: 6]。**同时，首页背景音乐 `home-bgm.mp3` 在 0.5 秒内平滑淡出并暂停**。
  * **邮箱入口**：点击右下角森林邮箱图标，触发点击音效，页面平箱淡出并跳转至【我的信箱页】，此时 `home-bgm.mp3` 保持在 25% 音量正常循环播放。

### 3.2 专注模式——全屏禅定页
* **页面入口**：用户点击首页“START”按钮并成功触发全屏后进入[cite: 6]。
* **视频与声音接力逻辑**：
  1. 进入全屏的瞬间，**专注页常驻沉浸式背景音乐 `assets/audio/zen-bgm.mp3` 开启循环播放，音量严格控制在 `0.25` (25%)**。
  2. 画面首先承接播放 `assets/videos/start-run.mp4`（仅放一次）[cite: 6]。
  3. 监听该视频的 `onended` 事件，在结束瞬间，**无缝无闪烁地替换视频源为全新完美循环版 `assets/videos/running-loop.mp4`**，开启 `loop` 属性展现小刺猬在追踪镜头下、两侧树木匀速向后倒退的无限飞奔画面。
* **极简倒计时**：位于屏幕右上角，浅灰色，透明度 30%，仅显示 “分 : 秒”（如 24 : 59），每秒递减且文本整体无左右抖动[cite: 6]。
* **防误触主动退出**：唯一退出方式为按下键盘 `ESC` 键[cite: 6]。触发后退出全屏、**关闭并重置 `zen-bgm.mp3`**，返回首页（返回首页后自动重新唤醒 25% 音量的 `home-bgm.mp3`）。
* **成功结束触发**：当倒计时归零（`00:00`）时，**立刻切断并暂停 `zen-bgm.mp3`**，自动无缝进入【成功结束页】。

### 3.3 专注模式终点——成功结束页 与 信件页
* **页面入口**：全屏禅定页倒计时归零后自动触发进入，无手动入口[cite: 6]。
* **视频与音频控制**：页面瞬间将视频源切换为全新完成的 `assets/videos/go-home.mp4` 并开始播放（仅播放一次）[cite: 6]。**【新增规则】**：由于该视频已内置了专属的治愈系通关音乐，**此时允许播放视频文件自带的音频轨，但音量在代码层必须严格限制在 `0.25` (25%)**。视频播放期间屏蔽用户的一切鼠标点击操作。
* **代码层手写信浮现与极致像素级排版**：
  * 监听视频的 `onended` 事件，视频保持定格在最后一帧（小屋全景静止画面）作为底图[cite: 6]。
  * HTML+CSS 构建的手写信羊皮纸卡片（`assets/images/paper-texture.png`）从屏幕中央缓缓淡入（时长 1.2 秒）[cite: 6]。
  * **【新增排版铁律】**：虚拟大画布尺寸为 **1440×900px**，中央羊皮纸尺寸锁定为 **722×632px**，物理坐标严格定位在 **X: 376px, Y: 94px**。信纸内文字展示必须严格遵循以下布局约束：
    1. **文字安全区**：信纸四周必须预留 **40px 边距（`padding: 40px;`）**，所有文字必须锁死在安全区内，不得超出信纸边界。文字无变形、无旋转、无遮挡，和旧纸张质感完美融合。
    2. **称呼区**：固定渲染文本 **“亲爱的自律之星:”**，字号 **28px**，颜色为深褐色（#4A3E3D），字体为手写楷体（'KaiTi'），左对齐，距离安全区上边界 40px、左边界 40px。
    3. **正文区**：采用多维矩阵算法动态生成的治愈文本，字号 **32px**，深褐色手写楷体，左对齐，位于称呼下方。段落间距（`margin-bottom`）固定为 **24px**，整体相对信纸左侧进行 **40px 的左缩进**。行高统一，通过限制包裹层 `max-height: 320px; overflow-y: auto;` 并隐藏滚动条确保长文本绝不穿帮。
    4. **落款区**：固定渲染文本 **“森林精灵敬上”**，字号 **28px**，深褐色手写楷体，右对齐，利用绝对定位牢牢钉在距离安全区**下边界 40px、右边界 40px** 的物理位置。
* **退出返回**：数据静默写入本地 LocalStorage[cite: 6]。用户点击弹窗关闭图标或页面任意空白处，整个结束页整体淡出，返回首页[cite: 6]。**返回首页的同时，必须重新唤醒播放首页背景音乐 `assets/audio/home-bgm.mp3`（音量 25%）**。

### 3.4 信箱页面
* **页面入口**：用户在首页点击右下角的森林邮箱图标进入[cite: 6]。
* **页面逻辑**：
  * 从本地 Web Storage 读取用户历史信件数据，按时间倒序排列展示[cite: 6]。右上角显示累计收集数量与总专注时长统计卡片[cite: 6]。
  * **【新增音效联动】**：在信箱页常驻播放 **`assets/audio/home-bgm.mp3`（音量 25%）**。用户在信箱页点击任何卡片、关闭按钮或返回箭头时，均触发 **`assets/audio/click.mp3`（音量 25%）**。
* **弹窗行为规范**：点击任意信件卡片弹出【单封手写信详情弹窗】[cite: 6]。点击弹窗右上角木质“X”图标关闭弹窗时，视口焦点必须精准返回「我的信箱页」母页，保留原页面的滚动位置，绝对不能直接跳回产品首页[cite: 6]。
* **空状态处理**：未收集任何信件时，中央显示 "还没有收到小刺猬的信哦，快去陪它回家吧～" 的柔和提示[cite: 6]。

## 5. 异常状态与全局边界处理

* **媒体资源加载失败 (后备防御)**：当 `<video>` 或 **`<audio>`** 标签触发 `onerror` 事件（例如视频/音频文件丢失或路径错误）时，系统需静默捕获该异常。视频底层容器自动显现出颜色编码为 `#3A4733` 的静态高斯模糊底色作为后备背景，且允许以静态图片平替显示[cite: 6]。**即使音频资产加载失败，核心倒计时与界面 UI 依旧正常渲染并保持精确运行，绝不阻塞主业务流**。
* **浏览器全屏权限受限（API被禁用）**：部分浏览器拒绝执行 `requestFullscreen`[cite: 6]。系统需捕获 Promise Rejection，静默降级为“网页内全屏”（通过 CSS 将禅定页容器的 `z-index` 提到最高，铺满整个当前标签页视口），确保计时与**音频切换**主流程不中断。
* **本地存储（LocalStorage）配额爆满**：使用 `try-catch` 包裹写入操作[cite: 6]。若捕获到 `QuotaExceededError` 异常，系统不显示报错弹窗，优先静默覆盖删除最早期（时间戳最老）的 1 封信件，确保本次专注成功的新信件能百分百顺利存入[cite: 6]。
